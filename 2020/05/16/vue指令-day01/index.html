2.vue指令

一.v-text、v.html、v-pre、v-once、v-cloak

  • 注意：（单向绑定）
    • 数据对象上的值改变，插值会发生变化
    • 当插值发生变化并不会影响数据对象的值

v-text单向绑定$\color{red}$

1.设置标签的文本值

  • v-text = data的属性名
  • v-text=”message”

2.拼接文本(在最前面接上)

  • v-text = data的属性名+插值表达式
  • v-text=”message+’!’”

3.v-text指令用于将数据填充到标签中，作用于插值表达式类似

  • 没有闪动问题
  • $\color{red}如果有HTML标签不会转换输出，输出的是纯文本$

v.html单向绑定

1.设置标签的innerHTML

  • 可能有安全问题, 一般只在可信任内容上使用 v-html，永不用在用户提交的内容上

2.$\color{red}将其当html标签解析后输出$

v-pre单向绑定

  • 显示原始信息跳过编译过程
  • 跳过这个元素和它的子元素的编译过程。
  • 一些静态的内容不需要编译加这个指令可以加快渲染

v-once单向绑定

  • 执行一次性的插值【当数据改变时，插值处的内容不会继续更新】
  • v-once的应用场景：如果显示的信息后续不需要再修改，可以提高性能。
  • 可以在浏览器中输入
    • app.msg=123来测试

    • 当数据发生变化的时候，视图也就发生变化
    • 当视图发生变化的时候，数据也会跟着同步变化

  • 限制在 <input>、<select>、<textarea>、components中使用

  • $\color{red}mvvm设计思想中的vm$

4.事件修饰符 (点’.’的指令后缀)

  • 1.在事件处理程序中调用（复杂）
    • event.preventDefault() 阻止冒泡
    • event.stopPropagation()阻止默认行为
  • 2.注意顺序，修饰符可以串联（简单,可以链式调用）
    • @click.stop=’xxx’ 阻止冒泡
    • @click.prevent=’xxx’ 阻止默认行为
    • @click.stop.prevent=”xxx” 阻止冒泡也阻止默认事件
    • @click.self=”xxx” 只当在 event.target 是当前元素自身时触发处理函数
  • 3.不同顺序的写法
    • @click.prevent.self 会阻止所有的点击
    • @click.self.prevent 只会阻止对元素自身的点击。
<!-- 阻止单击事件继续传播 -->
<a @click.stop="doThis"></a>

<!-- 提交事件不再重载页面 -->
<form @submit.prevent="onSubmit"></form>

<!-- 阻止冒泡也阻止默认事件 -->
<a @click.stop.prevent="doThat"></a>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div @click.self="doThat">...</div>

A、对冒泡的理解
若要阻止事件冒泡，只需将a标签中的@click 改成@click.stop 即可
这样点击a标签事件时不会触发外层div的事件

 <div  @click="click1()" >
             <a @click.stop="click2()"></a>
  </div>
---------------------------------
export default {
  name: "Home",
  data() {
    return {}
    },
    methods{
       click1(){
           alert('div事件')
       },
       click2(){
          alert('div内a标签的事件')
       },
    }
  }

B、对默认事件的理解
button元素绑定了一个contextmenu事件，单击鼠标右键会触发该事件，并调用show()方法
此时浏览器窗口不仅会出现alert弹框，还会出现浏览器默认的菜单选项
若要阻止默认行为，只需将@contextmenu 改成@contextmenu.prevent 即可 

<div >
    <button type="button" value="按钮" @contextmenu.prevent="show()">点我</button >
</div>

四、v-show、v-if（v-show性能更好）$\color{red}分支结构$

原理：

  • v-if是动态的向DOM树内添加或者删除DOM元素
    • v-if不停的销毁和创建
    • v-if切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件
  • v-show本质就是标签display设置为none，控制隐藏
    • v-show只编译一次，后面其实就是控制css

使用场景：

  • v-if根据表达值的true\false，切换元素的显示和隐藏（$\color{red}操纵DOM元素$）
    • $\color{red}是否渲染到页面$
    • 1- 多个元素 通过条件判断展示或者隐藏某个元素
    • 2- 多个元素，进行两个视图之间的切换
    • v-if、v-else-if、v-else
  • v-show根据表达值的true\false，切换元素的显示和隐藏（$\color{red}操纵style样式$）
    • $\color{red}已渲染到页面，是否显示$

使用：

1.直接v-show = true\false
2.间接v-show = 自定义值

  • data中定义：初始自定义值
  • methods中的函数内部：—自定义值= true\false
    重点：取反显示
    this.isShow = !this.isShow;

v-if第一种

<div id="app">
        <span v-if="flag">
           如果flag为true则显示,false不显示!
        </span>
</div>
<script>
    var vm = new Vue({
        el:"#app",
        data:{
            flag:true
        }
    })
</script>

v-if第二种

<div v-if="type === 'A'"> A </div>
<div v-else-if="type === 'B'"> B </div>
<div v-else-if="type === 'C'"> C </div>
<div v-else> Not A/B/C </div>
<script>
    new Vue({
      el: '#app',
      data: {
        type: 'C'
      }
    })
</script>

v-show

<div id="app">
      //v-show
      <input type="button" value="切换显示状态" @click="changeIsShow">
      <input type="button" value="累加年龄" @click="addAge">
      <img src="./img/monkey.gif" alt="" v-show="true">
      <img src="./img/monkey.gif" alt="" v-show="isShow">
      <img src="./img/monkey.gif" alt="" v-show="age>=18">

      //v-if
      <h2 v-if="temperature>=35">热死啦</h2>
</div>
<script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
<script>
      var app = new Vue({
        el:"#app",
        data:{
          isShow:true,
          age:15,
          temperature:40
        },
        methods: {
          changeIsShow:function(){
            this.isShow = !this.isShow;
          },
          addAge:function(){
            this.age++;
          }
        },
      })
</script>

五、v-for$\color{red}循环结构-遍历数组$

1.用于循环数组里面的值-可以是对象，也可以是普通元素
2.不推荐同时使用 v-if 和 v-for

  • 一起使用时，v-for 具有比 v-if 更高的优先级

3.v-bind:key(:key)

  • key来给每个节点做一个唯一标识,区分不同的元素
  • key的作用主要是为了高效的更新虚拟DOM
<ul>
  <li v-for="item in items" :key="item.id">{{item}}</li>
</ul>

第一种遍历数组

  • item 是数组里面的每一项
  • myFruits是 data中的数组名
  • key:id,区分id
<div id="app">
    <div>水果列表</div>
    <ul>
      <li v-for='item in fruits'>{{item}}</li>
      <li v-for='(item, index) in fruits'>{{item + '---' + index}}</li>
      <li :key='item.id' v-for='(item, index) in myFruits'>
        <span>{{item.ename}}</span>
        <span>-----</span>
        <span>{{item.cname}}</span>
      </li>
    </ul>
  </div>
  <script type="text/javascript" src="js/vue.js"></script>
  <script type="text/javascript">
    var vm = new Vue({
      el: '#app',
      data: {
        fruits: ['apple', 'orange', 'banana'],
        myFruits: [{
          id: 1,
          ename: 'apple',
          cname: '苹果'
        },{
          id: 2,
          ename: 'orange',
          cname: '橘子'
        },{
          id: 3,
          ename: 'banana',
          cname: '香蕉'
        }]
      }
    });
  </script>

第二种遍历对象

顺序固定，名字可以自定义

  • 第一个参数：v 代表对象的值
  • 第二个参数：k 代表对象的键
  • 第三个参数：i 代表索引

原生js遍历对象（原生不能遍历数组）

    var obj = {
      uname: 'lisi',
      age: 12,
      gender: 'male'
    }
    for(var key in obj) {
      console.log(key, obj[key])
    }
<div id="app">
    <div v-if='v==13' v-for='(v,k,i) in obj'>{{v + '---' + k + '---' + i}}</div>
</div>

 new Vue({
  el: '.app',
  data: {
    obj: {
        uname: 'zhangsan',

        gender: 'female'
    }
  }
})
</script>
