2.vue指令

一.v-text、v.html、v-pre、v-once、v-cloak

  • 注意：（单向绑定）
    • 数据对象上的值改变，插值会发生变化
    • 当插值发生变化并不会影响数据对象的值

v-text单向绑定$\color{red}$

1.设置标签的文本值

  • v-text = data的属性名
  • v-text=”message”

2.拼接文本(在最前面接上)

  • v-text = data的属性名+插值表达式
  • v-text=”message+’!’”

3.v-text指令用于将数据填充到标签中，作用于插值表达式类似

  • 没有闪动问题
  • $\color{red}如果有HTML标签不会转换输出，输出的是纯文本$

v.html单向绑定

1.设置标签的innerHTML

  • 可能有安全问题, 一般只在可信任内容上使用 v-html，永不用在用户提交的内容上

2.$\color{red}将其当html标签解析后输出$

v-pre单向绑定

  • 显示原始信息跳过编译过程
  • 跳过这个元素和它的子元素的编译过程。
  • 一些静态的内容不需要编译加这个指令可以加快渲染

v-once单向绑定

  • 执行一次性的插值【当数据改变时，插值处的内容不会继续更新】
  • v-once的应用场景：如果显示的信息后续不需要再修改，可以提高性能。
  • 可以在浏览器中输入
    • app.msg=123来测试

v-model双向数据绑定

  • 注意：双向数据绑定

    • 当数据发生变化的时候，视图也就发生变化
    • 当视图发生变化的时候，数据也会跟着同步变化

  • 限制在 <input>、<select>、<textarea>、components中使用

  • $\color{red}mvvm设计思想中的vm$

v-cloak

防止页面加载时出现闪烁问题

  • 1、通过属性选择器 选择到 带有属性 v-cloak的标签 让他隐藏

    • 元素隐藏：display: none;

  • 2、 让带有插值语法的添加 v-cloak 属性

    • <div v-cloak ></div>

  • 在 数据渲染完场之后，v-cloak 属性会被自动去除

    • v-cloak一旦移除就没有这个属性
    • 属性选择器就选择不到该标签，即对应的标签会变为$\color{red}可见$

4.事件修饰符 (点’.’的指令后缀)

  • 1.在事件处理程序中调用（复杂）
    • event.preventDefault() 阻止冒泡
    • event.stopPropagation()阻止默认行为
  • 2.注意顺序，修饰符可以串联（简单,可以链式调用）
    • @click.stop=’xxx’ 阻止冒泡
    • @click.prevent=’xxx’ 阻止默认行为
    • @click.stop.prevent=”xxx” 阻止冒泡也阻止默认事件
    • @click.self=”xxx” 只当在 event.target 是当前元素自身时触发处理函数
  • 3.不同顺序的写法
    • @click.prevent.self 会阻止所有的点击
    • @click.self.prevent 只会阻止对元素自身的点击。

注意：
A、对冒泡的理解
若要阻止事件冒泡，只需将a标签中的@click 改成@click.stop 即可
这样点击a标签事件时不会触发外层div的事件

B、对默认事件的理解
button元素绑定了一个contextmenu事件，单击鼠标右键会触发该事件，并调用show()方法
此时浏览器窗口不仅会出现alert弹框，还会出现浏览器默认的菜单选项
若要阻止默认行为，只需将@contextmenu 改成@contextmenu.prevent 即可

三、v-bind属性切换

响应地更新 HTML 属性
设置元素的属性（如src、title、class、href）

原始写法：
1.v-bind:src = “xxx属性名”
2.v-bind:title = “xxx”
3.v-bind:class = “xxx”
4.v-bind:href = “xxx”

简写：
1.:src = “xxx”
2.:title = “xxx”
3.:class = “xxx”
4.:href = “xxx”

1.属性绑定(有默认url，可以切换)

2.用v-bind和v-on实现v-model的双向绑定（两者等同，后者简单）

方法一：
<input type="text" v-bind:value="msg" v-on:input='handle'>

  • v-bind:value绑定input的value属性
  • v-on:input对应input事件
  • this.msg = event.target.value;(target对应input事件)

方法二：效果一样
<input type="text" v-bind:value="msg" v-on:input='msg=$event.target.value'>

方法三：效果一样
<input type="text" v-model='msg'>

3.绑定对象

  • :class绑定一个对象，动态切换class属性。（可以与普通的class特性共存）
    • 键为对应的类名
    • 值为对应data中的数据
  • 例子
    • 键textColor，textSize 对应的渲染到页面上的CSS类名
    • 值isColor，isSize 对应vue data中的数据（如果为true 则对应的类名渲染到页面上，当 isColor 和 isSize 变化时，class列表将相应的更新）

4.绑定class样式(里面还可以嵌套对象)

方式1：data里一个一个对应
<div v-bind:class='[activeClass, errorClass, {test: isTest}]'>测试样式</div>

  • activeClass: ‘active’,
  • errorClass: ‘error’,
  • isTest: true

方式2：数组对应
<div v-bind:class='arrClasses'></div>

  • arrClasses: ['active','error']

方式3：对象对应
<div class="base" v-bind:class='objClasses'></div>

  • objClasses: {active: true, error: true}

5.绑定style样式

方式1：一个一个直接定义
<div v-bind:style='{border: borderStyle, width: widthStyle, height: heightStyle}'></div>

方式2：对象定义一个样式
<div v-bind:style='objStyles'></div>

方式3：对象数组定义多个样式
<div v-bind:style='[objStyles, overrideStyles]'></div>

区别：

  • 绑定对象
    • 对象的属性（要渲染的类名）
    • 对象的属性值（ data 中的数据）
  • 绑定class数组
    • 数组里面存的是data 中的数据

3.绑定style

  • CSS 属性名
    • 1.用驼峰式或短横线分隔 (‘ab-cd’，用单引号括起来)
    • 2.将多个样式对象应用到同一个元素(数组形式)
    • 3.直接写内联样式

  • 默认isActive的初始值是false

  • 点击a，触发toggleActive函数，做取反切换成true
    • （再在：class中做三元判断，对应.active的样式）

四、v-show、v-if（v-show性能更好）$\color{red}分支结构$

原理：

  • v-if是动态的向DOM树内添加或者删除DOM元素
    • v-if不停的销毁和创建
    • v-if切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件
  • v-show本质就是标签display设置为none，控制隐藏
    • v-show只编译一次，后面其实就是控制css

使用场景：

  • v-if根据表达值的true\false，切换元素的显示和隐藏（$\color{red}操纵DOM元素$）
    • $\color{red}是否渲染到页面$
    • 1- 多个元素 通过条件判断展示或者隐藏某个元素
    • 2- 多个元素，进行两个视图之间的切换
    • v-if、v-else-if、v-else
  • v-show根据表达值的true\false，切换元素的显示和隐藏（$\color{red}操纵style样式$）
    • $\color{red}已渲染到页面，是否显示$

使用：

1.直接v-show = true\false
2.间接v-show = 自定义值

  • data中定义：初始自定义值
  • methods中的函数内部：—自定义值= true\false
    重点：取反显示
    this.isShow = !this.isShow;
