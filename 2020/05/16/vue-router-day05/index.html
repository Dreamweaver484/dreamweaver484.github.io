vue-route路由

  • 后端路由监听URL地址：根据不同的URL请求返回不同的内容【性能低】
    • URL请求地址与服务器资源的关系
    • 服务器根据不同的URL资源分发不同的资源
  • 前端路由监听用户事件：根据不同的用户事件来显示不同的页面内容【性能高】
    • 用户事件与事件处理函数之间的关系
    • 依靠hash值(锚链接)的变化进行实现

1.前端路由–简单例子

  • SPA（单页面应用程序）
    • 整个网站只有一个页面
    • 支持浏览器地址栏的前进和后退操作
    • 基于hash值的变化，并监听hash值变化的事件

点击页面中的菜单或者按钮改变URL的hash值，根据hash值的变化来控制组件的切换

  • 在页面中有四个超链接：
`<a href="#/zhuye">主页</a>`
`<a href="#/keji">科技</a>`
`<a href="#/caijing">财经</a>`
`<a href="#/yule">娱乐</a>`
  • 对应四个子组件
    // 4 个子组件
    // 主页组件
    const zhuye = {
        template: '<h1>主页信息</h1>'
    }
    // 科技组件
    const keji = {
        template: '<h1>科技信息</h1>'
    }
    // 财经组件
    const caijing = {
        template: '<h1>财经信息</h1>'
    }
    // 娱乐组件
    const yule = {
        template: '<h1>娱乐信息</h1>'
    }
  • 父组件绑定4个子组件
const vm = new Vue({
    el: '#app',
    data: {
    comName: 'zhuye'
    },
    // 注册私有组件
    components: {
        zhuye,
        keji,
        caijing,
        yule
    }
})

  • 当点击超链接时，会改变url地址中hash值

  • 当hash值被改变时，就会触发onhashchange事件

  • 根据hash值来让不同的组件进行显示：

  • 获取hash值

    • console.log(location.hash);

  • switch-case语句切换前端路由

window.onhashchange = function() {
    //slice（1）从下标为1的位置往后提取
    switch(location.hash.slice(1)){
        case '/zhuye':
            vm.comName = 'zhuye'
            break
        case '/keji':
            vm.comName = 'keji'
            break
        case '/caijing':
            vm.comName = 'caijing'
            break
        case '/yule':
            vm.comName = 'yule'
            break
    }
}

2.Vue Router

Vue Router依赖于Vue，所以需要先引入Vue，再引入Vue Router

Vue Router的特性：

  • 支持H5历史模式或者hash模式
  • 支持嵌套路由
  • 支持路由参数
  • 支持编程式路由
  • 支持命名路由
  • 支持路由导航守卫
  • 支持路由过渡动画特效
  • 支持路由懒加载
  • 支持路由滚动行为

1.Vue Router的简单使用

A.导入vue.js和vue-router.js文件

<script src="lib/vue_2.5.22.js"></script>
<script src="lib/vue-router_3.0.2.js"></script>

B.添加路由链接

  • <router-link>默认渲染为a标签
  • to属性 默认渲染href属性
  • to属性值 会被渲染为 # 开头的 hash地址

C.添加路由填充位（路由占位符）

  • 通过路由规则匹配到的组件，会渲染到router-view中
<router-link to="/user">User</router-link>
<router-link to="/login">Login</router-link>
<router-view></router-view>

D.定义路由子组件

var User = {
    template:"<div>This is User</div>"
    }
var Login = {
    template:"<div>This is Login</div>"
    }

E.配置路由规则并创建路由实例（父组件）

  • routes 属性 是路由规则数组
  • 数组中每一个路由规则都是一个对象，对象中至少包含path和component两个属性
    • path ： 路由匹配的hash地址
    • component ： 路由规则对应要展示的组件对象
var myRouter = new VueRouter({
    routes:[{
        path:"/user",
        component:User
        },
        {
        path:"/login",
        component:Login
        }]
})

F.将 路由实例 *挂载到 *Vue实例 中

  • router 属性 :挂载路由对象
    • 注：es6中如果属性和属性值一致，可以简写为一个
new Vue({
    el:"#app",
    //通过
    router:myRouter
})

2.Vue Router路由重定向

路由重定向：为页面设置默认展示的组件，在路由规则中添加一条redirect路由规则

  • routes 属性 是路由规则数组

  • 数组中每一个路由规则都是一个对象，对象中至少包含path和component两个属性

    • path ： 路由匹配的hash地址（页面最初始的地址）
    • component ： 路由规则对应要展示的组件对象
    • redirect ：路由重定向（要被重定向的新地址）

  • path 设置为/表示页面最初始的地址,

  • redirect 表示要被重定向的新地址，设置为一个路由即可

var myRouter = new VueRouter({
    //routes是路由规则数组
    routes: [{
            path:"/",
            redirect:"/user"
        },
        {  
            path: "/user",
            component: User
        },
        {
            path: "/login",
            component: Login
            }]
})

3.嵌套路由

当一级路由显示的组件中还有新的子级路由链接以及内容。

  • 如/index的路由（一级路由）
  • 则/index/a、/index/b 为嵌套的二级路由

A.在一级子组件中的 template 模板中添加 二级路由链接
B.在一级子组件中的 template 模板中添加 二级路由占位符

const Register = {
    template: `<div>
        <h1>Register 组件</h1><hr/>
        <!-- 二级子路由链接 -->
        <router-link to="/register/tab1">tab1</router-link>
        <router-link to="/register/tab2">tab2</router-link>
        <!-- 二级子路由的占位符 -->
        <router-view />
    <div>`
    }

C.定义二级路由子组件

//Register下的二级子组件
    const Tab1 = {
        template: '<h3>tab1 子组件</h3>'
    }
    //Register下的二级子组件
    const Tab2 = {
        template: '<h3>tab2 子组件</h3>'
    }

D.配置路由规则并创建路由实例（父组件）

  • 在一级路由规则中添加 children 数组
    • 数组中填写二级子路由规则
const router = new VueRouter({
    routes: [{
            path: '/register',
            component: Register,
             // children 数组表示子路由规则
            children: [{
                    path: '/register/tab1',
                    component: Tab1
                },
                {
                    path: '/register/tab2',
                    component: Tab2
                }]
           }]
    })

4.动态路由匹配【多个链接共用一个路由规则】

4.1【方式一：restful形式】

A.3个不同的路由链接，前缀一样，后缀参数不一样

B.使用一个共同的路由子组件

  • $route.params.id 可以拿到后缀参数
    • $route 是路由规则
const User = {
    template: '<h1>用户id为: {{$route.params.id}}</h1>'
}

C.配置路由规则并创建路由实例（父组件）

  • 获取路径传参的数据，3个路由链接 对应的路径，简化为 path: '/user/:id'
const router = new VueRouter({
        // 所有的路由规则
        routes: [
            { path: '/', redirect: '/user'},
            { path: '/user/:id', component: User },
        ]
    })

5.命名路由【给路由取别名】

在创建路由实例对象，及所有的路由规则时

  • 添加 name 属性,添加别名
var myRouter = new VueRouter({
    routes: [
        { path: "/user/:id", component: User, name:"user"},
    ]
})
  • 添加别名之后，使用别名跳转，并可传递参数
    • 方式一：更改原路由链接 to 为 绑定对象路由链接 :to
    • 方式二：编程式导航

方式一：

<router-link to="/user">User</router-link>

<!-- 原名字 -->
<router-link to="/user/1">User</router-link>

<!-- 命名别名后 -->
<router-link :to="{ name:'user' , params: {id:1} }">User</router-link>

方式二：

myRouter.push( { name:'user' , params: {id:123} } )

6. 页面导航

6.1 声明式导航：通过点击链接的方式实现导航

  • 普通网页
    • <a>xxx</a>

6.2 编程式导航：调用js的api方法实现导航

  • 普通网页
    • location.href
  • vue-router
    • this.$router.push(hash地址);
    • this.$router.go(n);

this.$router.push(“hash地址”);：

//字符串式的路径地址
this.$router.push("/login");
//对象式的路径地址
this.$router.push({ name:'user' , params: {id:123} });
//name属性的对象式路径地址
this.$router.push({ path:"/login" });
//带查询字符串的对象式路径地址，变成/login?username=jack
this.$router.push({ path:"/login",query:{username:"jack"} });

this.$router.go( n );
参数：n为数字，参考history.go

this.$router.go( -1 );
  • 在用户子组件中的methods中添加编程式导航api【push】
    • this.$router.push('/register') 跳转
  • 在注册子组件中的methods中添加编程式导航api【go】
    • this.$router.go(-1) 后退一步
//用户子组件
    const User = {
        props: ['id', 'uname', 'age'],
        template: `<div>
            <h1>用户id为: {{id}} - 姓名为:{{uname}} - 年龄为：{{age}}</h1>
            <button @click="goRegister">跳转到注册页面</button>
        </div>`,
        methods: {
            goRegister() {
                this.$router.push('/register')
            }
        }
    }
//注册子组件
    const Register = {
        template: `<div>
            <h1>Register 组件</h1>
            <button @click="goBack">后退</button>
                </div>`,
         methods: {
            goBack() {
                this.$router.go(-1)
            }
        }
    }
