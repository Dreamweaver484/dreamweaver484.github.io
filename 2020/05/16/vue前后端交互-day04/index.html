分析下PUT和PATCH的区别：
实例：假设我们有一个UserInfo，里面有userId， userName， userGender等10个字段。现在只修改userName。

【PUT是全局更新，后端不仅更新接收到的userName字段，还更新其他的9个字段。】
【PATCH是局部更新，后端仅更新接收到的userName字段。】

异步

  • JavaScript的执行环境是「单线程」

  • 单线程，指一次只能完成一项任务，这个任务执行完后才能执行下一个，它会「阻塞」其他任务。这个当前任务可称为主线程

  • 异步模式可以一起执行多个任务

  • JS中常见的异步调用

    • 1.定时任务
    • 2.ajax
    • 3.事件函数

4.promise异步简单实例

  • Promise(function(resolve, reject){...}

    • 第一个参数 resolve ：异步操作执行成功后的回调函数
    • 第二个参数 reject ：异步操作执行失败后的回调函数

  • resolve('xxx');

    • 执行成功后的结果

  • reject('xxx');

    • 执行失败后的结果

  • 获取结果方法一：

    • then 函数，在函数里面的参数是$\color{red}获取成功结果$，对应resolve函数
    • catch 函数，在函数里面的参数是$\color{red}获取失败结果$，对应resolve函数
promise.then((result) => {
    console.log(result);
})
.catch((err)=> {
    console.log(err);
})

  • 获取结果方法二：

    • 单独 then 方法 指定 resolved 状态和 reject 状态的回调函数
    • 第一个参数：获取成功结果
    • 第二个参数：获取失败结果
promise.then(function(data){
    console.log(data)
  },function(info){
    console.log(info)
  });

03.html

<script type="text/javascript">
//创建发送ajax请求的函数
  function queryData(url) {
  //  创建一个Promise实例
    var p = new Promise(function(resolve, reject){
      var xhr = new XMLHttpRequest();
      //不止调用一次，状态变化就调用
      xhr.onreadystatechange = function(){
        if(xhr.readyState != 4) return;
        if(xhr.readyState == 4 && xhr.status == 200) {
          // 处理正常的情况
          resolve(xhr.responseText);
        }else{
          // 处理异常情况
          reject('服务器错误');
        }
      };
      xhr.open('get', url);
      xhr.send(null);
    });
    return p;
  }

  queryData('http://localhost:3000/data')
    .then(function(data){
      console.log(data)
      return queryData('http://localhost:3000/data1');
    })
    .then(function(data){
      console.log(data);
      return queryData('http://localhost:3000/data2');
    })
    .then(function(data){
      console.log(data)
    });
</script>

6.promise 基本API

  • console.dir(Object);
    • 打印出该对象的所有属性和属性值.

$\color{red}实例方法:$

  • .then()
    • 得到异步任务执行成功的结果
  • .catch()
    • 得到异步任务执行失败的结果，获取异常信息
  • .finally()
    • 成功与否都会执行（不是正式标准）

$\color{red}静态方法$

  • Promise.all 接受一个数组作 参数对象（url）

    • 数组中的 对象 均为promise实例，并发处理多异步任务，在所有任务执行完成 才能得到结果

    • （如果不是一个promise，该项会被用Promise.resolve转换为一个promise)。它的状态由这三个promise实例决定

  • Promise.race 接受一个数组作 参数对象（url）

    • 数组中的 对象 均为promise实例，并发处理多异步任务，只要有一个任务执行完成 就能得到结果

    • 当p1, p2, p3中有一个实例的状态发生改变（变为fulfilled或rejected），p的状态就跟着改变。并把第一个改变状态的promise的返回值，传给p的回调函数

  • 把a1路由设置延时1秒

  • 把a2路由设置延时2秒

  • 把a3路由设置延时3秒

app.js

app.get('/a1', (req, res) => {
  setTimeout(function(){
    res.send('Hello TOM!')
  },1000);
})
app.get('/a2', (req, res) => {
  setTimeout(function(){
    res.send('Hello JERRY!')
  },2000);
})
app.get('/a3', (req, res) => {
  setTimeout(function(){
    res.send('Hello SPIKE!')
  },3000);
})
  • Promise.all([p1,p2,p3]).then(function(result){..})
    • Promise的all() 中的参数 [p1,p2,p3] 和 返回的结果一 一对应
  • Promise.race([p1,p2,p3]).then(function(result){..})
    • Promise的then() 只获得结果’P1’,因为p1执行较快。
    • p2,p3仍在继续执行，但执行结果将被丢弃。

二、fetch 基本API【ajax中xhr的升级api】

1.fetch

  • fetch会返回Promise实例
    • 是原生js，没有使用XMLHttpRequest对象

使用：

  • fetch(url, options)

  • 第一个参数 url ：请求的url地址 , 如 'http://localhost/books'

    • url 可以添加参数

  • 第二个参数 options ：请求参数对象

    • 指定对应的 method 请求使用的方法
    • 默认的是 GET 请求，其他POST，DELETE，UPDATE，PATCH，PUT
    • 指定对应的 headers 和 body 设置请求头
  • 【1.传统的URL传递参数】可以携带发送的参数，如id=123,
  • 在app.js中app方法：app.get
  • 在app.js中路由显示为：$\color{red}/books$
  • 在app.js中传递的参数显示为：$\color{red}req.query.id$
  • fetch中$\color{red}url$后缀添加 ? ：'http://localhost/books?id=123'
  • fetch中$\color{red}method$ 添加：method:'get'
  • 【2.Restful的URL传递参数】可以携带发送的参数，如id=456,
  • 在app.js中app方法：app.get
  • 在app.js中路由显示为：$\color{red}/books/:id$
  • 在app.js中传递的参数显示为：$\color{red}req.params.id$
  • fetch中$\color{red}url$后缀添加 / ：'http://localhost/books/456'
  • fetch中$\color{red}method$添加：method:'get'
  • 【3.delete传递参数】可以携带发送的参数，如id=789,
  • 在app.js中app方法：app.delete
  • 在app.js中路由显示为：$\color{red}/books/:id$
  • 在app.js中传递的参数显示为：$\color{red}req.params.id$
  • fetch中$\color{red}url$后缀添加 / ：'http://localhost/books/456'
  • fetch中$\color{red}method$添加 ：method:'delete'
  • 【4.post传递参数（简单body和headers）】可以携带发送的参数，如id=123,
  • 在app.js中app方法：app.post
  • 在app.js中路由显示为：$\color{red}/books/:id$
  • 在app.js中传递的参数显示为：$\color{red}req.params.id$、$\color{red}req.body.name$、$\color{red}req.body.pwd$
  • fetch中 $\color{red}url$ 后缀添加 / ：'http://localhost/books/123'
  • fetch中 $\color{red}method$ 添加 ：method:'post'
  • fetch中 $\color{red}body$ 添加 &连接 ：name=zhangsan&pwd=123
  • fetch中 $\color{red}headers$ 添加 对象形式：{ 'Content-Type': 'application/x-www-form-urlencoded' }
  • 【5.post传递参数（JSON的body和headers）】可以携带发送的参数，如id=123,
  • 在app.js中app方法：app.post
  • 在app.js中路由显示为：$\color{red}/books/:id$
  • 在app.js中传递的参数显示为：$\color{red}req.params.id$、$\color{red}req.body.name$、$\color{red}req.body.pwd$
  • fetch中 $\color{red}url$ 后缀添加 / ：'http://localhost/books/123'
  • fetch中 $\color{red}method$ 添加 ：method:'post'
  • fetch中 $\color{red}body$ 添加 &连接 ：JSON.stringify({ uname: '张三',pwd: '456' })
  • fetch中 $\color{red}headers$ 添加：{ 'Content-Type': 'application/json' }

注意：支持json
引入： const bodyParser = require('body-parser')
在app.js需加一句处理参数： app.use(bodyParser.json());

  • 【6.put传递参数（JSON的body和headers）】可以携带发送的参数，如id=123,
  • 在app.js中app方法：app.put
  • 在app.js中路由显示为：$\color{red}/books/:id$
  • 在app.js中传递的参数显示为：$\color{red}req.params.id$、$\color{red}req.body.name$、$\color{red}req.body.pwd$
  • fetch中 $\color{red}url$ 后缀添加 / ：'http://localhost/books/123'
  • fetch中 $\color{red}method$ 添加 ：method:'put'
  • fetch中 $\color{red}body$ 添加 &连接 ：name=zhangsan&pwd=123

注意：支持json

  • 【没有fetch中的methods，直接.get()或.post,其它一致】
  • axios.get( url , { params:{..} } )
  • axios.delete( url , { params:{..} } )
    • 第一个参数 url ：通过 传统url 以 ?id=123 的形式传递参数
    • 第一个参数 url ：通过 restful 以 /123 的形式传递参数
    • 第二个参数 参数对象 ：直接通过 $\color{red}params$ 以 对象 形式传递参数
  • post 和 put 请求传递参数
  • 方式一：axios.post( url , {..} ) 【没有params】
  • 方式二：var params = new URLSearchParams();
  • 方式二：再params.append(属性，属性值)添加传递参数
  • 方式二：axios.post( url , params)
    • 第一个参数 url ：通过 传统url 以 ?id=123 的形式传递参数
    • 第一个参数 url ：通过 restful 以 /123 的形式传递参数
    • 第二个参数 json对象 ：以 json 形式传递参数
    • 第二个参数 URLSearchParams ：以 json 形式传递参数

app.js

app.get('/axios', (req, res) => {
  res.send('axios get 传递参数' + req.query.id)
})
app.get('/axios/:id', (req, res) => {
  res.send('axios get (Restful) 传递参数' + req.params.id)
})
app.delete('/axios', (req, res) => {
  res.send('axios get 传递参数' + req.query.id)
})
app.post('/axios', (req, res) => {
  res.send('axios post 传递参数' + req.body.uname + '---' + req.body.pwd)
})
app.put('/axios/:id', (req, res) => {
  res.send('axios put 传递参数' + req.params.id + '---' + req.body.uname + '---' + req.body.pwd)
})

3.axios 响应结果

响应结果的主要属性：

  • data：实际响应回来的数据
    • xxx.data.xxx
  • headers:响应头信息
  • status：响应状态码
  • statusText：响应状态信息

app.js

app.get('/axios-json', (req, res) => {
  res.json({
    uname: 'lisi',
    age: 12
  });
})

//一定要有data属性才能拿到请求值

})

4.axios 全局配置【简化axios】

  • 配置请求的基准URL
    • axios.defaults.baseURL = 'xxx';
  • 配置 超时时间
    • axios.defaults.timeout = 'xxx';
  • 配置公共的请求头
    • axios.defaults.headers.common['Authorization'] = AUTH_TOKEN;
  • 配置公共的 post 的 Content-Type
    • axios.defaults.headers.post['Content-Type'] = 'application/x-www-form-urlencoded';
    • axios.defaults.headers.post['Content-Type'] = 'application/json';

12.html

<script type="text/javascript">
axios.defaults.baseURL = 'https://localhost:3000/';
axios.defaults.timeout = 2500;
axios.defaults.headers.common['Authorization'] = AUTH_TOKEN;
axios.defaults.headers.post['Content-Type'] = 'application/x-www-form-urlencoded';

//简化后的写法
axios.get('axios-json').then(function(ret){
      console.log(ret.data.uname)
    })
</script>

5.axios 拦截器【类似app.use】

  • 请求拦截器【在请求发送前进行一些操作】

    • 例如在每个请求体里加上token，之后更改容易

  • 响应拦截器【在接收到响应后进行一些操作】

    • 例如在服务器返回登录状态失效，需要重新登录的时候，跳转到登录页

  • 请求拦截器：axios.interceptors.request.use()

    • 第一个参数: function(config) {...}【信息配置】
      • 必须有 return config;
    • 第二个参数: function(err) {...}【错误处理】

  • 响应拦截器：axios.interceptors.response.use()

    • 第一个参数: function(res) {...}【信息配置】
      • 必须有 var data = res.data; return data;
    • 第二个参数: function(err) {...}【错误处理】

13.html

<script type="text/javascript">
    //请求拦截器
    axios.interceptors.request.use(function(config) {
      console.log(config.url)
      //添加请求头
      config.headers.mytoken = 'nihao';
      return config;
    }, function(err){
      console.log(err)
    })

    //响应拦截器
    axios.interceptors.response.use(function(res) {
       console.log(res)
       //获取到实际数据
      var data = res.data;
      return data;
    }, function(err){
      console.log(err)
    })

    axios.get('http://localhost:3000/adata')
    .then(function(data){
      console.log(data)
    })
  </script>

四、async、await和axios（es7）

1.async、await基本语法

  • async 作为一个关键字放到函数前面
    • 返回值是：$\color{red}一个promises实例对象$
  • await 关键字在使用async定义的函数中使用
    • await 后面跟一个Promise实例对象
    • 返回值是：$\color{red}异步执行的结果$

14.html

<script type="text/javascript" src="js/axios.js"></script>
<script type="text/javascript">

  //配置请求的基准URL
  axios.defaults.baseURL = 'http://localhost:3000/';

  //同步代码
  axios.get('adata')
  .then(function(ret){
    console.log(ret.data)
  })

  //异步代码【简单】
  async function queryData() {
    var ret = await axios.get('adata');
    return ret.data;
  }
  queryData().then(function(data){
    console.log(data)
  })

  //异步代码【promise实例对象】
  async function queryData() {
    var ret = await new Promise(function(resolve, reject){
      setTimeout(function(){
        // 执行成功
         resolve('nihao')
      },1000);
    })
    return ret;
  }
  queryData().then(function(data){
    console.log(data)
  })
</script>

2.多个async、await请求

  • 一个async函数 + 多个await请求
    • 一个await即一个异步处理的结果
    • 第一个异步请求：var info = await axios.get('async1');
    • 第二个异步请求：var ret = await axios.get('async2?info=' + info.data);
  • 在第二个异步请求的路由中，可以根据前一个异步请求返回的结果进行判断，是否执行下去【info是第二个的拼接参数，从第一个路由中拿到的】
    • if(req.query.info == 'hello') {...}

app.js

app.get('/async1', (req, res) => {
  res.send('hello')
})
app.get('/async2', (req, res) => {
  if(req.query.info == 'hello') {
    res.send('world')
  }else{
    res.send('error')
  }
})

  <script type="text/javascript" src="js/axios.js"></script>
  <script type="text/javascript">

    //配置请求的基准URL
    axios.defaults.baseURL = 'http://localhost:3000/';


    async function queryData() {
      var info = await axios.get('async1');
      var ret = await axios.get('async2?info=' + info.data);
      return ret.data;
    }

    queryData().then(function(data){
      console.log(data)
    })
  </script>
