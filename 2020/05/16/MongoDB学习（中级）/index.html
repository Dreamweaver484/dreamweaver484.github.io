10.mongoose

  • nodejs用对象的形式表示MongoDB的文档，即数据把文档转换为对象
    • 用中间件和应用逻辑挂钩
    • 创建schema的文档结构约束
    • Model对象是集合中的所有文档的表示，相当于MongoDB数据库中的集合
//引入
var mongoose = require("mongoose");

//连接数据库(端口号默认27017，可以省略不写)
//因为没有事务操作，所以只需要连接一次
//mongoose.connect('mongodb://数据库的ip地址：端口号/数据库名',{useMongoClient:true});
// mongoose.connect('mongodb://localhost:27017/test',{useMongoClient:true});

mongoose.connect('mongodb://localhost:27017/test');

var Schema = mongoose.Schema;

//创建Scheme（模式）对象，就是文档结构约束
var stuSchema = new Schema({
    name:String,
    age:Number,
    gender:{
        type:String,
        default:"male"
    },
    address:String
});

//通过Scheme来创建Model模型，就是来映射数据库的集合
//mongoose.model(modelName,Schema);
//modelName是数据库的集合名（表）
//注意：自动将集合名添加s（复数）

var StuModel = mongoose.model("student",stuSchema);


//向数据库插入文档
StuModel.create([
    {
        name:"zhangsan",
        age:20,
        gender:"famale",
        address:"HangZhou"
    },
    {
        name:"lisi",
        age:22,
        gender:"male",
        address:"ShangHai" 
    }
],function(err){
    if(!err){
        console.log("插入成功");
    }else{
        console.log("插入失败");
    }
});

//监听数据库的连接状态
mongoose.connection.once('open',function(){
    console.log("数据库连接成功");
    //在node控制台显示
    //console.log(arguments);
});

mongoose.connection.once('close',function(){
    console.log("数据库连接断开");
});

// //断开数据库连接（不常用）
mongoose.disconnect();

$\color{red}10.1创建、查询$

  • 1.向数据库插入文档
    • Model名.create(doc,function(err){});
  • 2.查询符合条件的第一个文档
    • Model名.findOne(conditions,projection,options,callback)
  • 3.查询所有符合条件的文档
    • Model名.find(conditions,projection,options,callback)
  • 4.根据文档的id属性查询文档
    • Model名.findById(id,projection,options,callback)
      • doc是要插入的文档,function(err){}是回调函数
      • conditions查询条件
      • projection投影
      • options查询选项（skip、limit）
      • callback查询结果通过回调函数返回
  • 5.还可以在Navicat中用db.students.find();查询，
  • $\color{red}查询结果都是文档对象，是Model的实例$

$\color{red}10.2更新$

  • 1.修改一个或多个文档
    • Model名.update(conditions,doc,options,callback);
  • 2.修改一个文档
    • Model名.updateOne(conditions,doc,options,callback);
  • 3.修改多个文档
    • Model名.updateMany(conditions,doc,options,callback);
  • 4.替换一个文档
    • Model名.replaceOne(conditions,doc,options,callback);
      • conditions修改条件
      • doc是修改后的文档对象
      • options修改选项（oldEnough、ferret、multi）
      • callback修改结果通过回调函数返回

11.用户

1.默认情况下，MongoDB 不会进行身份验证，也没有账号
2.超级用户和数据库用户。用户信息保存在 db.system.users
3.一个数据库可以包含多个用户，$\color{red}一个用户只能在一个数据库下，不同数据库中的用户可以同名$
4.特定数据库比如 test1 下的用户 test_user1，不能够访问其他数据库 test2，但是可以访问本数据库下其他用户创建的数据

一、添加用户
$\color{red}用户在哪个库创建，就要在哪个库auth认证，并在mongo登录时也要先连接认证库$

>use admin
>db.createUser({
  user:'test_user',
  pwd:'123456',
  customData:{"desc":"这是介绍"}，
  roles:[{role:'userAdminAnyDatabase',db:'admin'} | "<role>" ]
 })
//验证是否存在对应用户
> db.auth("test_user","password")
  • user用户名
  • pwd密码
  • customData介绍
  • roles
    • role用户的角色
    • db数据库
    • roles字段，指定内置角色和用户定义的角色

注：用户的角色:

    1. 数据库用户角色
    • read;允许用户读取指定数据库
    • readWrite; 允许用户读写指定数据库
    1. 数据库管理角色
    • dbAdmin:允许用户在指定数据库中执行管理函数，如索引创建、删除，查看统计或访问system.profile
    • dbOwner
    • userAdmin:允许用户向system.users集合写入，可以找指定数据库里创建、删除和管理用户
    1. 集群管理角色(只在admin数据库中可用)
    • clusterAdmin:赋予用户所有分片和复制集相关函数的管理权限
    • clusterManager
    • clusterMonitor
    • hostManager；
    1. 备份恢复角色
    • backup
    • restore
    1. 所有数据库角色(只在admin数据库中可用)
    • readAnyDatabase:赋予用户所有数据库的读权限
    • readWriteAnyDatabase:赋予用户所有数据库的读写权限
    • userAdminAnyDatabase:赋予用户所有数据库的userAdmin权限
    • dbAdminAnyDatabase :赋予用户所有数据库的dbAdmin权限
    1. 超级用户角色(只在admin数据库中可用)
    • root:赋予用户所有数据库的dbAdmin权限
    1. 内部角色
    • __system

二、修改用户

db.updateUser(
  "<username>",
  {
    customData : { 这是介绍 },
    roles : [
              { role: "readWriteAnyDatabase", db: "admin" } | "<role>",
              ...
            ],
    pwd: "123456"
    },
    writeConcern: {"xxx"})
  • username用户名
  • customData介绍
  • roles修改的角色们
    • role用户的角色
    • db数据库
  • pwd密码
  • writeConcern：用来指定mongod对写操作的回执行为

三、修改用户权限，不会覆盖原权限信息，只新增权限

  • db.grantRolesToUser(
    “jianlong”,
    [
    {
    role:'readWrite',
    db:'test'
}
    ]
    );

四、修改用户密码

  • db.changeUserPassword(“username”,”newPasswd”)

五、删除用户

  • db.dropUser(“username”)

六、查看角色的具体privilege（权限）

  • db.getRole(“SenderHis_Read”,{showPrivileges:true})

12.复制（最好使用逻辑DNS主机名而不是IP地址）

  • mongodb的复制至少需要两个节点。
  • 各个节点常见的搭配方式为：一主一从、一主多从。
    • 主节点，负责处理客户端请求，同步主节点的数据；
    • 其余的都是从节点，负责复制主节点上的数据
  • 副本集
    • N 个节点的集群
    • 任何节点可作为主节点
    • 所有写入操作都在主节点上

1.副本集各节点IP如下：

  • 172.16.250.238
  • 172.16.250.239
  • 172.16.250.240

2.依次修改各个节点的 $\color{red}mongodb.conf$ 配置文件，增加副本集相关配置:

dbpath=D:\mongoDB\data\db
logpath=D:\mongoDB\data\log
fork=true
logappend=true
bind_ip=172.16.250.238   # 此处填写服务器的IP(三个不同的写三次)
port=27017
replSet=rs0             # 设置副本集名称，在各个配置文件中，其值必须相同

3.在三个节点上执行,加载文件配置,启动MongoDB服务:
mongod -config D:\mongoDB\mongodb.conf

rs0:PRIMARY> for(var i=0;i<10000;i++){db.customer.insert({"name":"user"+i})}
输出：WriteResult({ "nInserted" : 1 })
rs0:PRIMARY> db.customer.count()
输出：10000


步骤二：在 Secondary 从节点上查看客户数据是否已经同步
rs0:SECONDARY> rs.slaveOk()
rs0:SECONDARY> db.customer.count()
输出：10000

9.副本集故障转移测试

步骤一：关闭Primar主节点的数据库
mongod --shutdown --dbpath D:\mongoDB\data

步骤二：查看Primary节点关闭之前的状态
rs0:PRIMARY> rs.isMaster()
rs0:PRIMARY> rs.status()

步骤三：登录任意其他节点上查看Primary节点关闭之后的状态
> mongo 172.16.250.239:27017
> rs.isMaster()
> rs.status()

步骤四：再次启动 172.16.250.238:27017 节点，由于其选举优先级最高，自动被选举为Primary
> mongo 172.16.250.238:27017
> rs.isMaster()
> rs.status()

13.MongoDB分片

  • Shard:(Shard Server)
    用于存储实际的数据块。一个shard server角色可由几台机器组个一个replica set承担，防止主机单点故障

  • Config Server:
    mongod实例。存储了整个 ClusterMetadata，其中包括 chunk信息。

  • Query Routers:（Route Process）
    前端路由。客户端由此接入，且让整个集群看上去像单一数据库，前端应用可以透明使用。

1.分片结构端口分布

Shard Server 1：27020
Shard Server 2：27021
Shard Server 3：27022
Config Server ：27100
Route Process：40000
步骤一：启动Shard Server
新建D:\mongoDB\shared（在里面新建s0、s1、s2、log文件夹）
bin目录下运行下列命令：
> mongod --port 27020 --dbpath "D:\mongoDB\shared\s0" --logpath "D:\mongoDB\shared\log\s0.log" --logappend --fork
> mongod --port 27021 --dbpath "D:\mongoDB\shared\s1" --logpath "D:\mongoDB\shared\log\s1.log" --logappend --fork
> mongod --port 27022 --dbpath "D:\mongoDB\shared\s2" --logpath "D:\mongoDB\shared\log\s2.log" --logappend --fork

步骤二： 启动Config Server
新建D:\mongoDB\shared（在里面新建config文件夹）
bin目录下运行下列命令：
> mongod --port 27100 --dbpath "D:\mongoDB\shared\sconfig" --logpath "D:\mongoDB\shared\log\config.log --logappend --fork

步骤三： 启动Route Process
【chunkSize这一项是用来指定chunk的大小的，单位是MB，默认大小为200MB】
> mongos --port 40000 --configdb localhost:27100 --fork --logpath "D:\mongoDB\shared\log\route.log" --chunkSize 500

步骤四： 配置Sharding
登录到mongos，添加Shard节点
> mongo admin --port 40000

mongos> db.runCommand({ addshard:"localhost:27020" })
输出：{ "shardAdded" : "shard0000", "ok" : 1 }
...(一共添加10个)
mongos> db.runCommand({ addshard:"localhost:27029" })
输出：{ "shardAdded" : "shard0009", "ok" : 1 }


#设置分片存储的数据库
mongos> db.runCommand({ enablesharding:"test" })
输出{ "ok" : 1 }
mongos> db.runCommand({ shardcollection: "test.log", key: { id:1,time:1}})
输出：{ "collectionsharded" : "test.log", "ok" : 1 }


步骤五：将数据库连接接入接口40000

数据备份

-h：MongDB所在服务器地址:端口号
-d：需要备份的数据库

>mongotop

15.DBRefs数据库引用

{ $ref : , $id : , $db :  }
$ref：  集合名称
$id：   引用的id
$db:    数据库名称，可选参数

实例：（3.0版本）
>var user = db.users.findOne({"name":"Tom Benzamin"})
>var dbRef = user.address
>db[dbRef.$ref].findOne({"_id":(dbRef.$id)})


实例：（4.0版本）
>var user = db.users.findOne({"name":"Tom Benzamin"})
>var dbRef = user.address
>db[dbRef.$ref].findOne({"_id":ObjectId(dbRef.$id)})
