表单基本操作

1.获取单选框中的值（一个默认值）
每一个单选框必须要有value属性，且value 值不能一样

  • $\color{red}通过v-model双向绑定,类似input的name属性$
    • v-model=’gender’
  • data中，与v-model联动，是默认值
    • data: {gender: 2}

2.获取复选框中的值（一个数组）
每一个复选框必须要有value属性，且value 值不能一样

  • data 中的 xxx 我们要定义成数组，否则无法实现多选
    • v-model=’hobby’
  • data中，与v-model联动，是默认值
    • hobby: [‘2’, ‘3’]

3.获取下拉框和文本框中的值
每一个 option 必须要有value属性，且value 值不能一样

  • 需要给 select 通过 v-model 双向绑定一个值，multiple 多选
    • <select v-model='occupation' multiple>
  • 当某一个option 选中的时候 v-model 会将当前的 value值 改变 data 中的 数据,使用实时监控occupation的值，有默认值
    • occupation: [‘2’, ‘3’]

v-model表单修饰符

  • $\color{red}Vue无法将字符串转换成数值$
    • 当开始输入非数字的字符串时，属性值将实时更新成相同的字符串。
    • 即使后面输入数字，也将被视作字符串。
  • .number 【转换为数值】
    • 字符串转数值
  • .trim 【自动过滤用户输入的首尾空白字符，算长度可理解】
    • 只能去掉首尾的空格
  • .lazy 【将input事件切换成change事件】
    • 延迟了同步更新属性值的时机。
    • 将原本绑定在 input 事件的同步逻辑转变为绑定在 change 事件上

.lazy注意点：

  • change事件是 在 失去焦点 或者 按下回车键 才触发
  • input事件是 每次输入内容 都触发

.lazy使用场景：

  • 在用户输入信息，离开焦点时，立即做一个信息有无在数据库中注册的验证

自定义指令

注意：

  • 使用自定义的指令，在对用到的元素中，加上’v-‘的前缀
  • 在自定义指令中 若$\color{red}用普通方式定义$，如：
    • Vue.directive(‘focus’,function(){})
    • 则通过 v-focus 来使用
  • 在自定义指令中 若$\color{red}用驼峰命名的方式定义$，如：
    • Vue.directive(‘focusA’,function(){})
    • 则只能通过 v-focus-a 来使用

涉及指令钩子函数

  • bind：只调用一次
    • 指令第一次绑定到元素时调用。可以进行一次性的初始化设置。
  • inserted
    • 被绑定元素插入父节点时调用(仅保证父节点存在，但不一定已被插入文档中)
  • update
    • 所在组件的 VNode 更新时调用，但是可能发生在其子 VNode 更新之前。指令的值可能发生了改变，也可能没有
  • componentUpdated
    • 指令所在组件的 VNode 及其子 VNode 全部更新后调用。
  • unbind：只调用一次
    • 指令与元素解绑时调用

指令钩子函数【directive】传入参数

  • el：指令所绑定的元素，可以用来直接操作 DOM。
  • binding：一个对象
    • name：指令名，不包括 v- 前缀。
    • value：指令的绑定值
      • 例如：v-my-directive=”1 + 1” 中，绑定值为 2。
    • oldValue：指令绑定的前一个值，
      • 仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
    • expression：字符串形式的指令表达式。
      • 例如 v-my-directive=”1 + 1” 中，表达式为 “1 + 1”。
    • arg：传给指令的参数，可选。
      • 例如 v-my-directive:foo 中，参数为 “foo”。
    • modifiers：一个包含修饰符的对象。
      • 例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }。
  • vnode：Vue 编译生成的虚拟节点。
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用

1.Vue.directive 全局

  • 注册全局指令（使用时不带参数）
    • Vue.directive(‘focus’, {…}
      • 注册一个全局自定义指令 v-focus
    • inserted: function (el){}
      • 钩子函数inserted
      • 当绑定元素插入到 DOM 中。 其中 el为dom元素
    • el.focus();
      • 聚焦元素
    • <input type="text" v-focus>
      • 使用
  • 注册全局指令（使用时带参数）
    • bind :声明周期, 只调用一次，指令第一次绑定到元素时调用。
      • el :当前自定义指令的DOM元素
      • binding :通过自定义属性传递过来的值,存到 binding.value 里面

2.new Vue({.. directives:{..} ..}) 局部指令

  • 定义在 directives 的选项中
  • $\color{red}局部指令只能在当前组件里面使用$
  • $\color{red}当全局指令和局部指令同名时以局部指令为准$
  • 使用方法同全局指令

计算属性 computed

介绍

  • computed 适合对多个变量或者对象进行处理后返回一个结果值
    • 多个变量中的某一个值发生了变化，则监控的这个值也就会发生变化
  • watch 中的属性定义 和 methods 中的属性定义 平级

计算属性与方法的区别

  • computed中计算属性的方法是 基于响应式依赖进行缓存
    • 当多次调用 reverseString 的时候，只要里面 num 值不改变，就可以返回之前已经计算过的结果，
    • 直到data 中的 num 值改变 计算属性才会重新发生计算
  • methods中方法 不缓存
    • 调用methods中的方法 reverseMessage() 的时候，他每次会重新调用计算

使用

  • computed: {…}
    • split(‘’)分割
    • reverse()翻转
    • join(‘’)连接
  • 一定要有return 否则调用 reverseString 的时候无法拿到结果

侦听器 watch

  • 响应数据变化，用于异步或者开销较大的操作
  • $\color{red}watch 中的属性 一定是data 中 已经存在的数据$
    • watch 中的属性定义 和 methods 中的属性定义 平级
    • watch 中的属性名（函数名）与 data 的数据名 一致
  • 当需要监听一个对象的改变时，普通的watch方法无法监听到对象内部属性的改变，只有data中的数据才能够监听到变化，此时就需要deep属性对对象进行深度监听

简单使用：

  • watch中的 firstName 对应着data 中的 firstName
  • watch中的 lastName 对应着data 中的 lastName
  • 当 firstName 值 改变的时候 会自动触发 watch
  • 当 firstName 值 改变的时候 会自动触发 watch

1.watch的deep属性

  • Vue 深度监听对象或者对象指定属性
  • 监听器会一层层的往下遍历，给对象的所有属性都加上这个监听器，但是这样性能开销就会非常大了，任何修改obj里面任何一个属性都会触发这个监听器里的 handler
  • evlist是一个对象，val监听到变化后的值，oldVal是变化前的值。

  • immediate:true
    • 如果在 wacth 里声明了 firstName 之后，就会立即先去执行里面的handler方法
  • immediate:false
    • false就跟我们以前的效果一样，不会在绑定的时候就执行。

过滤器 filter(s)

  • filters 属性定义和 methods 平级
  • filters支持级联操作
    • 用‘|’管道符，同时调用多个filter函数
  • filters不改变真正的data，只改变渲染的结果，并返回过滤后的版本
  • $\color{red}过滤器中一定要有返回值$
  • 局部过滤器是filters，有s
    • 只有在本组件中使用
  • 全局过滤器是filter，没有s
    • 所有组件都可以使用

使用场景：

  • 1.双花括号插值
  • 2.v-bind表达式
  • 3.可自定义过滤器，用于文本格式化

调用方法：

  • {{msg | upper | lower}}
    • 第一个必须是：绑定值名
    • 第二个及以后是：过滤器函数名
    • 从左到右调用
  • upper 为局部过滤器（在data中的filters中）
  • lower 为全局过滤器（在Vue.filter(‘lower’, function(val) {..}中）

以下：

  • upper 定义为接收单个参数的过滤器函数
    • 表达式msg 的值将作为参数传入到函数中。
  • 然后继续调用，定义为接收单个参数的过滤器 lower
    • 将upper 的结果传递到lower中
  • charAt(0)
    • 拿到一个字符
  • toUpperCase()、toLowerCase()
    • 转换大写、小写
  • slice(1)
    • 从第二个字符截取到最后

一、过滤器未传递参数

二、过滤器中传递参数(定义的function中加参数)

  • 第一个参数不是传递的，对应管道符前的值

  • 第二个及以后的参数用来传递，对应管道符后函数的参数

  • filterA 被定义为接收三个参数的过滤器函数

    • 第一个参数:message 的值
    • 第二个参数:普通字符串 ‘arg1’
    • 第三个参数:表达式 arg2 的值

三、过滤器案例：格式化日期

转换为yyyy-MM-dd的形式

  • 必须有返回值
    • return value;

转换为yyyy-MM-dd hh:mm:ss的形式(正则匹配)

  • 必须有返回值（这里是返回函数）
    • return dateFormat(value, arg);

生命周期

  • 事物从出生到死亡的过程
  • Vue实例从创建 到销毁的过程伴随一些函数的自调用。
    • 这些函数为钩子函数

钩子函数

beforeCreate 在实例初始化之后，数据观测和事件配置之前被调用 此时data 和 methods 以及页面的DOM结构都没有初始化 什么都做不了
created 在实例创建完成后被立即调用此时data 和 methods已经可以使用 但是页面还没有渲染出来
beforeMount 在挂载开始之前被调用 此时页面上还看不到真实数据 只是一个模板页面而已
mounted el被新创建的vm.$el替换，并挂载到实例上去之后调用该钩子。 数据已经真实渲染到页面上 在这个钩子函数里面我们可以使用一些第三方的插件
beforeUpdate 数据更新时调用，发生在虚拟DOM打补丁之前。 页面上数据还是旧的
updated 由于数据更改导致的虚拟DOM重新渲染和打补丁，在这之后会调用该钩子。 页面上数据已经替换成最新的
beforeDestroy 实例销毁之前调用
destroyed 实例销毁后调用
  • 挂载（初始化相关属性）
    • beforeCreate
    • created
    • beforeMount
    • mounted
  • 更新（元素或组件的变更操作）
    • beforeUpdate
    • updated
  • 销毁（销毁相关属性）
    • beforeDestroy
    • destroyed

数组变异方法（修改原有数据，会影响到原始数据）

  • 直接修改对象属性的值无法触发响应式。
    • 直接修改对象属性的值，只有数据改变，但是页面内容不变
  • 变异：原有功能不变的前提下对其进行功能拓展
push() 往数组最后面添加一个元素，成功返回当前数组的长度
pop() 删除数组的最后一个元素，成功返回删除元素的值
shift() 删除数组的第一个元素，成功返回删除元素的值
unshift() 往数组最前面添加一个元素，成功返回当前数组的长度
splice() 有三个参数，第一个是想要删除的元素的下标（必选），第二个是想要删除的个数（必选），第三个是删除 后想要在原位置替换的值
sort() sort() 使数组按照字符编码默认从小到大排序,成功返回排序后的数组
reverse() reverse() 将数组倒序，成功返回倒序后的数组

替换数组（生成新数组，不会影响原始数据）

filter filter() 方法创建一个新的数组，新数组中的元素是通过检查指定数组中符合条件的所有元素。
concat concat() 方法用于连接两个或多个数组。该方法不会改变现有的数组
slice slice() 方法可从已有的数组中返回选定的元素。该方法并不会修改数组，而是返回一个子数组
  • 变异方法：会影响数组的原始数据的变化。
  • 替换数组：不会影响原始的数组数据，而是形成一个新的数组（$\color{red}所以需要重新赋值$）。

动态数组响应式数据（应对索引操作没有响应式）

全局(让触发视图重新更新一遍，数据动态起来)

  • Vue.set(a,b,c)
    • a是要更改的数据(数组名称)
    • b是数据的第几项(数组索引)
    • c是更改后的数据(数组值)

局部(vm是组件)

  • vm.$set(a,b,c)
    • a是要更改的数据(数组名称)
    • b是数据的第几项(数组索引)
    • c是更改后的数据(数组值)

对比

1.处理数组索引：

  • vm.list[1] = 'lemon';
    • 直接更改，页面不会改变
  • Vue.set(vm.list, 1, ‘lemon’);
    • 全局
  • vm.$set(vm.list, 1, ‘lemon’);
    • 局部

2.处理对象属性（例子中添加了一个属性）

  • Vue.set(vm.info, ‘gender’, ‘female’);
    • 全局
  • vm.$set(vm.info, ‘gender’, ‘female’);
    • 局部
