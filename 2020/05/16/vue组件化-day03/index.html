组件可以扩展 HTML 元素，封装可重用的代码

一个组件的 data 选项必须是一个函数【注意点】

没有组件

data: {
  count: 0
}

组件中

  • 因此每个实例可以维护一份被返回对象的独立的拷贝
data: function () {
  return {
    count: 0
  }
}

组件注册

1.全局注册

  • Vue.component(‘组件名称’, { })
    • 第1个参数：标签名称
    • 第2个参数：模板template
  • 全局组件注册后，任何vue实例都可以用
<div id="example">
  <my-component></my-component>
</div>
<script>
    //   注册组件
    // 1、 my-component 就是组件中自定义的标签名
  Vue.component('my-component', {
      template: '<div>A custom component!</div>'
    })
    // 创建根实例（也是组件）
    new Vue({
      el: '#example'
    })
</script>

全局组件注意点1:基础组件的自动化全局注册

  • 使用 require.context 只全局注册这些非常通用的基础组件

在应用入口文件 (比如 src/main.js) 中全局导入基础组件的示例代码:

import Vue from 'vue'
import upperFirst from 'lodash/upperFirst'
import camelCase from 'lodash/camelCase'

const requireComponent = require.context(
  // 其组件目录的相对路径
  './components',
  // 是否查询其子目录
  false,
  // 匹配基础组件文件名的正则表达式
  /Base[A-Z]\w+\.(vue|js)$/
)

requireComponent.keys().forEach(fileName => {
  // 获取组件配置
  const componentConfig = requireComponent(fileName)

  // 获取组件的 PascalCase 命名
  const componentName = upperFirst(
    camelCase(
      // 获取和目录深度无关的文件名
      fileName
        .split('/')
        .pop()
        .replace(/\.\w+$/, '')
    )
  )

  // 全局注册组件
  Vue.component(
    componentName,
    // 如果这个组件选项是通过 `export default` 导出的，
    // 那么就会优先使用 `.default`，
    // 否则回退到使用模块的根。
    componentConfig.default || componentConfig
  )
})

2.局部注册

步骤一：定义局部组件模板template

  • data属性：值
  • template属性：模板调用data的返回值
var HelloTom = {
      data: function(){
        return {
          msg: 'HelloTom'
        }
      },
      template: '<div>{{msg}}</div>'
    };

步骤二：当前注册它的vue实例中引入模板

  • vue实例中的components属性
    • 第一个属性：重新命名的短横线模板名
    • 第二个属性：引入定义好的局部组件模板名，如HelloTom
var vm = new Vue({
      el: '#app',
      data: {
      },
      components: {
        'hello-tom': HelloTom,
        'hello-jerry': HelloJerry
      }
    });

综合

    <hello-jerry></hello-jerry>
  </div>

    var HelloTom = {

        }

局部注册注意点1

  • 只能在当前注册它的vue实例中使用
  • 局部注册的组件在其子组件中不可用
    • 例如，如果你希望 局部组件ComponentA 在 局部组件ComponentB 中可用，则你需要这样写：
var ComponentA = { /* ... */ }

var ComponentB = {
  components: {
    'component-a': ComponentA
  },
  // ...
}

babel语法

import ComponentA from './ComponentA.vue'

export default {
  components: {
    ComponentA  //ComponentA: ComponentA 的缩写
    //即这个变量名同时是：用在模板中的自定义元素的名称;包含了这个组件选项的变量名
  },
  // ...
}

局部注册注意点2：在模块系统中局部注册

在一个假设的 ComponentB.js 或 ComponentB.vue 文件中：

import ComponentA from './ComponentA'
import ComponentC from './ComponentC'

export default {
  components: {
    ComponentA,
    ComponentC
  },
  // ...
}
//现在 ComponentA 和 ComponentC 都可以在 ComponentB 的模板中使用了。

3.组件注意事项

  • 1.组件参数的data值必须是函数
    • 同时函数要求返回一个对象
  • 2.组件模板必须是单个根元素
    • 如果不是，外层可添加一个div包裹
  • 3.组件模板内容可以是模板字符串
    • 用反引号 `` 来包裹
  • 4.组件命名方式
    • 驼峰式（全大写）Vue.component(‘HelloWorld’, {…})
    • 短横线（全小写）Vue.component(‘hello-world’, {…})
  • 5.组件使用
    • 驼峰式（全大写）
    • 短横线（全小写）
  • 6.规则(推荐短横线)
    • 在字符串模板中，可用驼峰式、短横线
    • 在普通的标签模板中，只用短横线，不能驼峰式
  <div id="app">
    <!-- 不能显示 -->
    <HelloWorld></HelloWorld>
    <!-- 可以显示 -->
    <hello-world></hello-world>
  </div>
<script type="text/javascript">
     Vue.component('HelloWorld', {
      data: function(){
        return {
          msg: 'HelloWorld'
        }
      },
      template: '<div>{{msg}}</div>'
    });
    Vue.component('hello-world', {
      data: function(){
        return {
          count: 0
        }
      },
      template: `
        <div>
          <button @click="handle">点击了{{count}}次</button>
          <button>测试</button>
          <HelloWorld></HelloWorld>
        </div>
      `,
      methods: {
        handle: function(){
          this.count++;
        }
      }
    })
    var vm = new Vue({
      el: '#app',
      data: {
      }
    });
  </script>

C、两种试图改变一个 prop 的情形

  • 第一种：这个 prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用
    • 解决：定义一个本地的 data 属性并将这个 prop 用作其初始值
props: ['initialCounter'],
data: function () {
  return {
    counter: this.initialCounter
  }
}
  • 第二种：这个 prop 以一种原始的值传入且需要进行转换数据
    • 解决：定义一个计算属性computed
props: ['size'],
computed: {
  normalizedSize: function () {
    return this.size.trim().toLowerCase()
  }
}

    propB: [String, Number],
    // 必填的字符串

    },
    // 带有默认值的数字
    propD: {
      type: Number,

    // 带有默认值的对象

2.子组件向父组件传值

a.在子组件Vue.component(‘menu-item’, {…});的template属性中

  • template的模板内用$emit()触发事件
    • 第一个参数:自定义的事件名称
    • 第二个参数:需要传递的数据

<button @click='$emit("enlarge-text", 5)'>扩大父组件中字体大小</button>

b.父组件用 v-on 监听子组件的事件（携带$event 参数）

    • 父级组件就会接收该事件并更新FontSize 的值。
  • 这里 handle 是Vue实例中 methods 对应的函数
  • 这里 $event 参数是指代该enlarge-text事件

<menu-item :parr='parr' @enlarge-text='handle($event)'></menu-item>

        handle: function(val){
          // 扩大字体大小

4.组件插槽（复用性）

插槽作用：
a.<slot></slot>父组件向子组件传递内容
b.<template slot='xxx'></template>把多条文本填充到插槽中

1.匿名插槽

  • <slot> 插槽元素（在子组件的template模板中使用）
  • 在html页面渲染时，子组件标签中的内容放置在插槽中

子组件：（插槽内容）

  • 子组件标签中$\color{red}有内容$<alert-box>有bug发生</alert-box>
    • slot会被该内容替换
  • 子组件标签中$\color{red}没有内容$<alert-box></alert-box>
    • 使用 slot 中的默认值

子组件模板：（插槽定义）

  • slot $\color{red}没有默认值$ <slot></slot>
  • slot $\color{red}有默认值$ <slot>默认内容</slot>
Vue.component('alert-box', {
      template: `
        <div>
          <strong>ERROR:</strong>
          <slot></slot>
        </div>
      `
    });

2.具名插槽（有名字的插槽）

  • <slot> 中的 “name” 属性绑定元素
  • template临时的包裹标签最终不会渲染到页面上
  • 具名插槽渲染顺序，取决于模板，不取决于父组件中元素的顺序

子组件：（插槽内容）

  • 子组件标签中$\color{red}有内容，有name属性$
    • 标签内有<template slot='xxx'>的属性
<base-layout>
      <template slot='header'>
        <p>标题信息1</p>
        <p>标题信息2</p>
      </template>
</base-layout>
  • 子组件标签中$\color{red}有内容，没有name属性$
    • 标签内没有<template slot='xxx'>的属性
<base-layout>
      <p>主要内容1</p>
      <p>主要内容2</p>
</base-layout>

子组件模板：（插槽定义）

  • slot name='xxx' $\color{red}没有默认值，有name属性$ 
<header>
    <slot name='header'></slot>
</header>
  • slot $\color{red}没有默认值，没有name属性$ 
<main>
    <slot></slot>
</main>

3.作用域插槽

  • 父组件对子组件加工处理
  • 在父组件中复用子组件的slot，又可以使slot内容不一致
  • 数据遍历由子组件定义，样式遍历由父组件定义

a.父组件中

  • :list属性绑定的方式传递实例data中list数据
  • 再通过子组件的props插槽接收数据
    • <fruit-list :list='list'></fruit-list>
    • props: ['list']

b.处理子组件【数据遍历由子组件定义）】

  • 模板定义（这里插槽提供了内容）
    • 绑定slot的info
      <slot :info='item'></slot>
  • 插槽可提供一个默认内容
    • 如果父组件没有为插槽提供内容，会显示默认的内容。
    • 如果父组件为插槽提供了内容，则默认的内容会被替换掉。
template: `
        <div>
          <li :key='item.id' v-for='item in list'>
            <slot :info='item'>{{item.name}}</slot>
          </li>
        </div>
      `

c.在父组件中得到子组件数据（slot-scope对象属性）

  • 使用template模板的固定slot-scope对象属性，命名的值自定义
    • <template slot-scope='slotProps'>
  • 如果想改变第3个slotProps的info.id的样式【样式遍历由父组件定义】，用v-if指令
  • 其余不变，用v-else指令
<fruit-list :list='list'>
  <template slot-scope='slotProps'>
      <strong v-if='slotProps.info.id==3' class="current">
            {{slotProps.info.name}}
      </strong>
      <span v-else>{{slotProps.info.name}}</span>
  </template>
</fruit-list>

综合

  <div id="app">
    <fruit-list :list='list'>
      <template slot-scope='slotProps'>
          <strong v-if='slotProps.info.id==3' class="current">
            {{slotProps.info.name}}
         </strong>
        <span v-else>{{slotProps.info.name}}</span>
      </template>
    </fruit-list>
  </div>

          </li>
        </div>
